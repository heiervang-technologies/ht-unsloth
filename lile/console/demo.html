      What the endpoints do
      
      
      chat — POST /v1/chat/completions. OpenAI-shaped. Extra lile.* block carries response_id and commit_cursor. Set after_commit_token to block generation until a specific training commit has landed.
      
      
      feedback — POST /v1/feedback. binary thumbs up/down → KTO. nl_critique → CoH with only a critique. nl_critique_with_rewrite → CoH with critique + better_response.
      
      
      objectives at a glance:
        sft / ntp / coh are Razin-safe (pure likelihood-up — see lile/GLOSSARY.md).
        kto / hinge / ccpd_v2 are preference-style and can exhibit likelihood displacement under adversarial pairs.
      
      
      
      Chat SFT vs NTP — Chat SFT wraps (prior turns, target) through the tokenizer's chat template and gradients only the assistant target. NTP tokenizes the raw text with no template and gradients every token. Use NTP for imprinting a passage/style; use Chat SFT for teaching a response to a message.
      
      
      train — POST /v1/train. Submits a batch of samples for one objective. Returns commit_token you can wait for.
      
      
      merge — POST /v1/state/merge. Folds the active LoRA adapter into the residual Δ and resets the adapter. Frees the adapter for new gradients without losing prior updates.
      
      
      snapshot — POST /v1/state/snapshot/save|load, GET /v1/state/snapshots. Persists or restores the residual + adapter + trajectory.
      
      
      trajectory — GET /v1/state/trajectory/tail?n=.... Canonical append-only log of every committed train step, merge, and feedback event.
      
      
      wait — POST /v1/wait?token=...&timeout=.... Block until a training commit token has landed. Useful for "train then generate with the new weights."
